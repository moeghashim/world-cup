# v0.3.1 Mobile UX QA

Date: 2026-06-08

Branch: `fix/v0.3.1-mobile-ux`

## Scope

- Fix the shared mobile `SiteHeader` overflow at mobile widths.
- Keep the page CTA reachable inside the mobile dropdown.
- Tighten homepage prize imagery and `/brackets` mobile panels so primary pages
  have no horizontal document scroll.
- Add automated Playwright mobile coverage for `/`, `/pickem`, `/brackets`,
  `/hosts`, `/profile`, and `/h/does-not-exist`.

## Red Test Evidence

Before the layout fix, `npm run test:e2e` failed 12/12 tests. The homepage at
360px reported `documentElement.scrollWidth = 534` against an expected maximum
of `361`, and the same overflow appeared across all shared-header pages.

## Green Test Evidence

After the layout fix, `npm run test:e2e` passed 12/12 tests across 360px and
390px mobile widths. The automated assertions cover:

- no horizontal document/body scroll;
- burger visible and fully inside the viewport;
- dropdown opens on tap;
- all seven primary nav links visible and within the viewport;
- mobile menu CTA visible and within the viewport;
- tapping a nav link navigates and closes the menu.
